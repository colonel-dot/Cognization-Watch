bridge.app

子女端应用的启动与全局初始化层，负责 Application 级别的配置与第三方 SDK 的初始化，只处理进程生命周期相关事务，不承载任何具体业务逻辑。

bridge.base

子女端 MVP 架构的基础抽象层，定义 View、Presenter 以及 Activity / Fragment 的通用结构与生命周期绑定规则，仅用于统一开发范式，不包含任何业务语义。

bridge.ui

子女端所有界面相关代码所在层，负责界面展示、用户交互与状态渲染，将用户操作转交给 Presenter，自身不参与业务决策。

bridge.presenter

子女端的业务协调层，负责组织数据获取、业务规则判断与能力调用，决定界面状态变化，是 MVP 架构中承载核心流程控制的中枢。

bridge.domain

子女端的业务规则与决策模型层，用于描述风险评估、行为分析和位置判断等核心逻辑，不依赖具体数据来源或界面实现。

bridge.data

子女端的数据访问层，统一封装网络请求、本地存储与数据映射，对外提供稳定的数据获取接口，不包含业务含义。

bridge.feature

子女端的能力调度层，对外暴露视频通话、定位监控、AR 辅助等功能入口，负责协调底层能力的启停与状态管理，而不关心业务规则。

bridge.integration

外部系统与 SDK 的适配层，用于隔离 Unity、第三方 SDK 与业务代码之间的差异，承担跨系统交互与回调转换的职责。

bridge.model

子女端内部使用的数据模型定义层，仅用于承载结构化数据，不包含业务逻辑或流程控制。

bridge.util

子女端通用工具集合，存放与具体业务无关的通用功能实现，为各层提供基础支撑。
